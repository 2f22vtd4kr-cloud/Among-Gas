> **The Node.js Buffer Quirk:** In your server-side snippet, you have new DataView(data.buffer). In Node.js, the data object returned by the ws library is a standard Node Buffer. While data.buffer exists (referencing the underlying ArrayBuffer), Node often allocates buffers from a shared pool. Reading data.buffer directly can sometimes point to a massive chunk of unrelated memory instead of just your packet.
> *The Fix:* Keep DataView exclusively for the HTML5 browser client. On your Node server, stick directly to the native Buffer methods like data.readUint8(0) and data.readInt16LE(1). It is faster and perfectly safe from pooling bugs.

# AMONG US TELEGRAM SPEC (MODULE 2: AUTHENTIC ROLE & DELTA SYNC ENGINE)
## 1. Authoritative Role Assignment Algorithm (0x12)
To prevent "Impostor spoofing" hacks, the client must never choose or request a role. When the host sends the 0x12 Game Start packet, the server executes a deterministic shuffle.
### Server Role Assignment Logic (TypeScript)
```typescript
function assignRoles(players: Map<number, Player>, impostorCount: number) {
  const playerIds = Array.from(players.keys());
  
  // Fisher-Yates Shuffle
  for (let i = playerIds.length - 1; i > 0; i--) {
    const j = Math.floor(Math.random() * (i + 1));
    [playerIds[i], playerIds[j]] = [playerIds[j], playerIds[i]];
  }

  // Assign Impostors based on the shuffled array index limits
  playerIds.forEach((id, index) => {
    const player = players.get(id);
    if (player) {
      player.role = index < impostorCount ? 'impostor' : 'crewmate';
      player.alive = true;
    }
  });
}

```
### Information Asymmetry Payload Enforcement
The server must **NEVER** broadcast the complete player array to everyone at match start.
 * **Crewmate Serialization:** If a player is a Crewmate, their start packet contains only *their* identity details. All other players are masked as "crewmate" in their local client state.
 * **Impostor Serialization:** If a player is an Impostor, their start packet includes the identities of the *other* Impostors so they can recognize their teammates on screen.

 3. **The 0xFF Compact Multi-Player Frame:** If three out of ten players moved past the threshold, the server packs *only* those three moving players into a single tight binary payload, ignoring the stationary players completely.
### Binary Layout Structure for Combined Move Deltas
 * **Byte 0:** 0xFF (Delta Sync Frame Type)

### Geometric Validation Routine
```typescript
function validateKill(attacker: Player, victim: Player, maxKillRange: number): boolean {
  if (!attacker.alive || attacker.role !== 'impostor') return false;
  if (!victim.alive || victim.role === 'impostor') return false; // No team killing

  // Calculate strict Euclidean distance vector on server values
  const dx = attacker.pos.x - victim.pos.x;
  const dy = attacker.pos.y - victim.pos.y;
  const separationDistance = Math.sqrt(dx * dx + dy * dy);

  return separationDistance <= maxKillRange;
}

```
